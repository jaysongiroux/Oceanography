Objective

Create a computer program that allows oceanographers and researchers to enter data on a user friendly UI that will parse the data (manually inputted or CSV File) and output the visualized data into an image that can be easily interpreted by the general public and other researchers.

Code Base:

The entirety of this toolkit was written on top of the electron framework using a combination of nodeJS libraries and react. While there are plenty of frameworks and languages that could have been used to create this UI, electron and NodeJS were chosen due to the familiarity and comfortability. Each team member has professional experience working on this framework making syntax and libraries easy to adjust to and optimize.

Expectation vs Reality

When Nick and Jason first began working on this final lab project they had initially created a large amount of requirements and tools that we would be able to implement. However, after surpassing the minimum hours required for this project weeks before it was due, the group began to realize in order to achieve the results they wanted it would take an extra semester worth of time.
The toolkit group decided to cut down requirements and deliver two useful, quality and reliable tools instead of an array of tools that were thrown together and not tested properly. This results in the toolkit only containing two tools: wave classification and data mapping.

DataTypes and visualization:

At the beginning of this project, the group agreed having a tool that would allow the user to input recorded data into a grid and have the option to visualize the data would be an extremely useful tool. The group began implementing a tool formerly known as “salinity/Depth Mapping Tool”. This tool continued to receive many revisions and additional features were added. This tool was no longer for just salinity and depth data. It could be used for any measurement that needed to be visualized. This tool allows the user to customize the table in a dynamic and intuitive way and be able to generate an interactive graph that they can adjust, move and capture for their documents and research papers.
The user needs to understand the importance of the graph types used in this tool. Heat maps are used for 2D visualization while surface plots are used for 3D visualization. It would make no sense to use a surface plot to plot salinity, however it would make sense to use a heat map for map salinity.
